# Pochi

思考の速度でポンチ絵を描くための、vimライクなモーダル作図ツール。

- **エディタ本体**: React + TypeScript + Vite (`app/`) — ブラウザ単体でも動くので、そのままweb展開できる
- **デスクトップシェル**: WPF + WebView2 (`desktop/`) — ネイティブのファイルダイアログを postMessage ブリッジで提供
- **公開先**: [GitHub Pages](https://sk0ya.github.io/Pochi/) — `main` へのpushで `.github/workflows/deploy-pages.yml` が `app/` をビルドして自動デプロイ

## データ形式

- **保存形式(正)**: 独自JSON `.pochi.json` — 図形と「どの図形同士が矢印で接続されているか」という意味情報を保持する
- **テーマ**: 編集画面はライト/ダークの2テーマ(既定はダーク)。ツールバー右の ☀ Light / 🌙 Dark トグルか `:theme [light|dark]` で切替(localStorageに保存)
- **エクスポート**: SVG(`:svg`)— 画面と同じテーマで書き出す(WYSIWYG)。資料に貼るときなど、その回だけテーマを変えたい場合は `:svg light` / `:svg dark` と引数で上書きできる
- **PNGコピー**: `:png` / `Ctrl+Alt+C` — SVGエクスポートと同じ見た目を2倍解像度でラスタライズし、OSクリップボードにコピー(選択中があればその範囲のみ)。テーマはSVGと同様
- **URL共有**: `:share` — 図形データをdeflate圧縮しbase64url化して `#d=<payload>` としてURLに埋め込み、OSクリップボードにコピー(失敗時はプロンプト表示)。そのURLを開くと起動時にlocalStorageの自動保存より優先して読み込まれ、以降はそのハッシュを消してそれが作業中のドキュメントになる(画像を含む図形は大きくなりがちで、ペイロードが約8000文字を超える場合は警告付きでコピーする)
- 編集中は localStorage に自動保存(次回起動時に復元。ただし `:share` のURLで開いた場合はそちらが優先)
- **最近使ったファイル**: デスクトップ版のみ、ツールバーの Open 横の ▾ から直近8件を開ける(パスをlocalStorageに記憶。Web版はブラウザのファイル選択がユーザー操作なしに再オープンできないため対象外)

SVGを正データにしなかったのは、接続情報・グリッドなどの意味情報の往復保持が難しくなるため。

## 共同編集(P2P)

`:collab`(またはツールバー右の 👥 ボタン)でルームを作成し、`#room=<id>` 付きのURLがクリップボードにコピーされる。そのURLを開いた人が同じ図面をリアルタイムに同時編集できる。`:collab off`(または 👥 ボタン再クリック)で退出。

- **完全P2P・サーバーレス**: [trystero](https://github.com/dmotz/trystero) を使い、公開Nostrリレー経由でWebRTCのシグナリングだけを行い、以降の編集データはピア間のWebRTCデータチャネルを直接流れる。自前サーバー不要なのでGitHub Pagesのままで動く(ルームIDが実質の鍵で、シグナリングもルームID由来のパスワードで暗号化される)
- **同期方式**: 図形・矢印ごとのLast-Writer-Wins(Lamportクロック+ピアIDタイブレーク)。同じ図形を同時に動かした場合はどちらか一方に全員が収束する。削除はトゥームストーンで管理し、z順も配列単位でLWW同期する(`app/src/collab/sync.ts`、テストあり)
- **参加時**: URLで参加すると、ルーム内の既存ピアから図面スナップショットを受信して自分のドキュメントを置き換える(置き換え前のドキュメントはアンドゥで復元可能)。参加者のローカル編集中データがルームに漏れることはない
- **プレゼンス**: 他の参加者のカーソルが色付きで表示され、ツールバーの 👥 に人数が出る
- **注意**: リロードすればハッシュから同じルームに再参加する。TURNサーバーは使っていないため、両者が対称NAT下にいる等の一部のネットワーク構成では接続できないことがある。ルームの図面はどこにも保存されない(各participantのlocalStorage自動保存のみ)

## ビルド & 起動

デスクトップ版は既定で公開ビルド([GitHub Pages](https://sk0ya.github.io/Pochi/))を読み込むので、フロントエンドをローカルでビルドする必要はない。`dotnet run` だけで起動する(ネイティブのファイルダイアログ等を提供する WebView2 ブリッジは、読み込み元が公開URLでも同じように動く)。

```powershell
cd desktop
dotnet run
```

### フロントエンドの読み込み元(優先順)

| 条件 | 読み込み元 | 用途 |
| --- | --- | --- |
| `POCHI_DEV_URL=http://localhost:5173` | ローカルの Vite 開発サーバー | フロントを触りながら HMR 開発 |
| `POCHI_LOCAL=1` | ローカルにバンドルした `wwwroot` | オフライン / 未pushの変更を確認 |
| (既定) | 公開ビルド | 通常利用。ビルド不要 |

公開ビルドに繋がらない(オフライン等)場合は、自動でローカルバンドルにフォールバックする。

- **開発時**: `cd app && npm run dev` で Vite を起動し、`POCHI_DEV_URL=http://localhost:5173` を設定してデスクトップ版を起動すると HMR で開発できる。ブラウザで直接 `http://localhost:5173` を開いてもよい。
- **オフライン用のローカルバンドル**: `dotnet build -p:BundleFrontend=true` で `npm run build` が走り、`app/dist` を `wwwroot` に固める。以降 `POCHI_LOCAL=1` で起動すればオフラインでも動く。

## キーバインド(vimモード)

モーダル設計。ステータスバーに現在のモードが表示される。`?` でヘルプ。

### NORMAL

| キー | 動作 |
| --- | --- |
| `h j k l` / 矢印 | カーソル移動(カウント可: `5l`。`Shift`併用で大きめステップ) |
| `r` / `e` / `q` / `g` | 四角 / 楕円 / ひし形 / 三角形を描く → DRAW モードへ |
| `o` | フレーム(コンテナ)を描く → DRAW モードへ。ラベルは左上に表示。フレームを移動すると、移動開始時点で中心がフレームの内側にある図形も一緒に動く(入れ子フレームも対応。枠線と左上のラベル部分だけがクリック対象で、内側は透過するので中の図形を直接クリックできる。右クリックメニューの「塗り」をONにすると内側にうっすら色がつくが、それでもクリックは透過したまま) |
| `a` | 矢印。図形の上で押すと**図形に接続**(移動に追従する) |
| `t` | テキスト作成 → INSERT |
| `i` | カーソル下の図形・矢印のテキスト編集(なければテキスト作成) |
| `v` | カーソル下の図形を移動(MOVE モード) |
| `s` | 図形をリサイズ(RESIZE モード)。複数選択中はまとめて拡大縮小 |
| `d` / `x` | カーソル下の図形・矢印を削除(図形を消すと接続矢印も消える) |
| `y` / `p` | ヤンク / カーソル位置にペースト |
| `u` / `Ctrl+r` | アンドゥ / リドゥ |
| `.` | 直前の編集を繰り返す(vimのドットリピート)。図形作成(`r`/`e`/`q`/`g` → Enter)・テキスト作成(`t`)・ペースト(`p`)はカーソル位置に、削除(`d`/`x`)はカーソル下のアイテムに対して。リサイズ・移動・矢印・ラベル編集は対象外。カウント指定(`3.`)は非対応 |
| `Enter` | カーソル下の図形を選択 |
| `f` | EasyMotion風のヒントジャンプ(HINT モード)。図形にラベルが表示され、そのキーを押すとカーソルが移動して選択(26個まで1文字、それ以上は2文字) |
| `m{a-z}` | カーソル位置をマーク(a〜z)に記録。図形ではなく座標を覚えるので、その図形を消してもマークは残る |
| `'{a-z}` | マークへジャンプ。未設定なら`mark not set: a`と表示。`z`と同様、画面中央に表示する |
| `z` | カーソルを画面中央に |
| `+` / `-` | ズーム |
| `Ctrl+A` | 全選択 |
| `Ctrl+G` | グループ化 / グループ解除(トグル) |
| `Ctrl+]` / `Ctrl+[` | 選択をひとつ前面へ / 背面へ |
| `:` | コマンドライン |
| `/` | ラベル検索。Enterで一番近い一致にジャンプして選択(図形・矢印ラベルを大小無視の部分一致、Escでキャンセル) |
| `n` / `N` | 直前の検索の次 / 前の一致へジャンプ(末尾・先頭で循環) |
| `?` | ヘルプ |

### DRAW / MOVE / RESIZE / ARROW(トランジェントモード)

`hjkl` で調整(カウント可)、`Enter` またはクリックで確定、`Esc` でキャンセル。

### INSERT

図形上にテキストエリアが出る。改行可。`Esc` で確定(IME対応)。

### コマンド

| コマンド | 動作 |
| --- | --- |
| `:w [name]` | 保存(デスクトップ版はダイアログ、2回目以降は上書き) |
| `:o` / `:e` | 開く |
| `:new` | 新規(ツールバーの **New** ボタンでも可) |
| `:svg [light\|dark]` / `:export svg [light\|dark]` | SVGエクスポート(引数なしは画面テーマで書き出し、引数でその回だけ上書き) |
| `:png [light\|dark]` / `:export png [light\|dark]` | PNG画像としてクリップボードにコピー(選択中があればその範囲のみ、失敗時はdiagram.pngをダウンロード。テーマはSVGと同様) |
| `:share` | 図形データを圧縮・エンコードしたURL(`#d=...`)をクリップボードにコピー。そのURLを開くと自動保存より優先して読み込む |
| `:theme [light\|dark]` | 画面テーマ切替(引数なしはトグル。ツールバー右の ☀/🌙 ボタンでも可) |
| `:vim off` / `:vim on` | vimモード切替(ツールバーのボタンでも可) |
| `:q` / `:wq` | 終了 |

### マウス操作(vimモードON/OFF共通)

- **✏ Autoツール(デフォルト): 手描きストロークを自動判定**
  - 閉じた丸っぽい形 → 楕円、閉じた角ばった形 → 四角、開いた線 → 矢印
  - 線の始点・終点が図形の上なら接続矢印(図形に追従)
  - 図形を認識すると、そのままテキスト編集状態になる(すぐ文字入力できる)
  - Autoツール中の図形移動は、クリックで選択してからドラッグ
- 図形ツール選択時: 空白をドラッグ = その図形をラバーバンドで描く(vimモードOFF時はそのままテキスト編集へ)
- Arrowツール中は図形から図形へドラッグで接続矢印
- Textツール中はクリックでテキスト作成
- 図形をドラッグ = 移動、右下の角ハンドル = リサイズ(複数選択時はまとめて拡大縮小)、辺の中央ハンドル = その方向だけ伸縮(反対側の辺は固定)、クリック = 選択
- 図形ドラッグ中は他図形との辺・中央揃えにスナップしてガイド線を表示
- グループ化した図形はクリック1つで全体が選択される(`Ctrl+G` でグループ化/解除)
- ▢ Frame ツールでドラッグするとフレーム(コンテナ)を描ける。中に図形を置いてフレームごと動かすと、中の図形も追従する
- 🖼 Image ボタンで画像ファイルを選んでカーソル位置に配置
- 📋 PNG ボタン / `Ctrl+Alt+C` で選択中の図形(なければ全体)をPNG画像としてクリップボードにコピー(2倍解像度、白背景。失敗時はdiagram.pngをダウンロード)。vimモードON/OFFどちらでも、テキスト編集中を除き動作
- ダブルクリック: 空白 = テキストボックスを作成して編集 / 図形 = テキスト編集
- **右クリック: コンテキストメニュー** — テキスト編集・コピー・複製・削除・最前面/最背面へ(1段階のみの移動も可)・グループ化/解除・文字サイズ(S/M/L)・色(7色パレット)。2つ以上選択中は整列(左右/上下揃え・中央揃え)、3つ以上選択中は等間隔配置(横/縦)も。コネクタ選択時は経路(直線/直角)切替・ベンドポイント追加/全削除も。空白での右クリックは貼り付け
- パン = **中ボタンドラッグ / Space+ドラッグ / ホイール**、ズーム = Ctrl+ホイール
- ツールバーの **⤢ Fit** で全体を画面に収める、ステータスバーの `100%` 表示をクリックでズームリセット
- `Ctrl+C` / `Ctrl+V` / `Ctrl+D` でコピー / 貼り付け / 複製(vim・マウスどちらのモードでも動作)
- vimモードOFF時は `Delete` 削除、`Ctrl+Z/Y`、`Ctrl+S` 保存、F2 テキスト編集

## 構成

```
app/src/
  model/     図形・接続のデータモデル、ジオメトリ、SVGシリアライズ
  collab/    P2P共同編集: sync.ts = アイテム単位LWWマージ(純粋ロジック), session.ts = trysteroルーム管理
  state/     reducer = モーダル入力ステートマシン(vimの中核)+ undo履歴
  input →    キーはApp.tsxのグローバルリスナーで捕捉し reducer にKEYアクションで流す
  components Canvas(SVG描画+マウス), StatusBar, Toolbar, TextEditOverlay, HelpOverlay
  bridge.ts  WebView2 postMessage ブリッジ(web時はダウンロード/file inputにフォールバック)
desktop/     WPFシェル。app/dist を仮想ホスト(https://app.pochi)にマップして表示
```
